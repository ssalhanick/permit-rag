# Session 2026-06-30 — Cognito smoke-test debug + deploy prep

## What was accomplished

- Debugged end-to-end Google SSO flow locally. Resolved 6 sequential issues:
  1. `global is not defined` — Vite polyfill for `amazon-cognito-identity-js` (already fixed prior session).
  2. CORS on 500 responses — added catch-all `@app.exception_handler(Exception)` in `api/main.py`; added Vite dev proxy (`/auth/me`, `/query`, etc. → `localhost:8000`) so all local API calls are same-origin.
  3. OAuth callback intercepted by Vite proxy — narrowed proxy rule from `/auth` to `/auth/me` only.
  4. `Token verification failed: Invalid audience` — `python-jose` requires `audience=` param when token has `aud` claim. Resolved by removing audience verification (`verify_aud: False`) since RS256 signature + issuer already prove token origin.
  5. `column "cognito_sub" does not exist` — `run_migration.py` was silently dropping SQL statements whose chunks started with comment lines (the `--` filter was too aggressive). Fixed to strip comment lines per-chunk rather than skip the whole chunk. Migration 013 re-run successfully.
  6. `COGNITO_APP_CLIENT_ID env var is required` — env var wasn't loading reliably on Windows (last-line-no-newline dotenv issue + `.env.local` missing the var). Added to `.env.local`, added trailing newline to `.env`, and ultimately removed the hard requirement by dropping audience verification.

- Created `frontend/.env.production` with public Cognito vars for CI/CD `npm run build`.
- Fixed `scripts/run_migration.py` to handle multi-statement SQL correctly (split + strip comments).
- Made `013_cognito_auth.sql` idempotent (`ADD COLUMN IF NOT EXISTS`, `CREATE INDEX IF NOT EXISTS`).
- Confirmed Google SSO flow works end-to-end locally (token exchange → `/auth/me` → user provisioned).
